# 运行环境规范

这份规范用于区分本地开发、Docker 本地测试和生产部署。以后启动项目前先看这里，避免端口、数据库、Redis 和 Docker 模式混在一起。

## 结论

本项目固定使用三种运行模式：

| 场景 | 推荐模式 | 说明 |
|------|----------|------|
| 日常开发调试 | 前端本机源码 + Docker 服务 | 前端 Vite 在本机跑，API 默认经 `3780`/Nginx 转容器；需要调后端源码时显式切换到 `8000` |
| 本地完整容器测试 | Docker Compose local | 本机用源码构建 api/web 镜像，完整模拟容器部署 |
| 生产环境 | 纯 Docker Compose | web、api、postgres、redis 全部在 Docker 内运行 |

不要在同一个场景里混用多个端口或多个 Redis/PostgreSQL 容器。

## 本地开发调试

本地开发模式用于改代码、调页面、看接口日志。

推荐调试结构：

```text
client npm run dev      -> http://localhost:5173
Vite proxy /api         -> http://localhost:3780
web container nginx     -> internal api:8000
PostgreSQL Docker       -> localhost:5433
Redis Docker            -> localhost:6380
```

默认情况下，`5173` 的 `/api`、`/static`、`/uploads` 会代理到 `3780`，再由 Nginx 转给容器内 API。这样本地调试和生产入口保持一致，避免只验证了 `3780` 却忘记 `5173` 代理链路。

只有在本机直接运行后端源码时，才显式切换 Vite 代理：

```bash
VITE_DEV_PROXY_TARGET=http://localhost:8000 npm run dev -- --host localhost --port 5173
```

只有直接运行本机后端源码时，后端使用 [server/.env](../server/.env)：

```env
DATABASE_URL=postgresql://modeluser:modelpass@localhost:5433/3dparthub
REDIS_URL=redis://localhost:6380
```

启动顺序：

```bash
colima start
docker compose -f docker-compose.local.yml up -d --build

cd client
npm run dev -- --host localhost --port 5173
```

检查状态：

```bash
docker ps
lsof -nP -iTCP:5433 -sTCP:LISTEN
lsof -nP -iTCP:6380 -sTCP:LISTEN
curl http://localhost:3780/api/health
curl http://localhost:5173/api/health
curl http://localhost:5173/
CHECK_VITE=1 node scripts/check-local-stack.mjs
```

停止本机前端时，停止对应 `npm run dev` 进程即可；如果单独启动了本机后端源码，也停止对应进程。数据库和 Redis 可以保留运行，避免频繁冷启动。

## 本地完整容器测试

本地完整容器测试用于确认 Docker 镜像构建、Nginx、容器网络和生产形态是否正常。

运行结构：

```text
web container      -> http://localhost:3780
api container      -> internal api:8000
PostgreSQL Docker  -> internal postgres:5432, host localhost:5433
Redis Docker       -> internal redis:6379, host localhost:6380
```

启动：

```bash
colima start
docker compose -f docker-compose.local.yml up -d --build
```

访问：

```text
http://localhost:3780
```

查看日志：

```bash
docker compose -f docker-compose.local.yml logs -f api
docker compose -f docker-compose.local.yml logs -f web
docker compose -f docker-compose.local.yml logs -f postgres
docker compose -f docker-compose.local.yml logs -f redis
```

停止但保留数据：

```bash
docker compose -f docker-compose.local.yml down
```

停止并删除本地测试数据：

```bash
docker compose -f docker-compose.local.yml down -v
```

只有明确要清空本地数据库和文件时才能使用 `down -v`。

## 生产环境

生产环境使用纯 Docker Compose，不使用 `npm run dev`，不使用 Vite 开发服务。

运行结构：

```text
web container
api container
postgres container
redis container
```

容器内部连接：

```env
DATABASE_URL=postgresql://modeluser:密码@postgres:5432/3dparthub
REDIS_URL=redis://redis:6379
```

生产环境对外入口默认是：

```text
http://服务器IP:3780
```

部署：

```bash
docker compose up -d
```

检查：

```bash
docker compose ps
docker compose logs api --tail 80
docker compose logs postgres --tail 80
docker compose logs redis --tail 80
curl http://localhost:3780/api/health
```

生产环境不要使用：

```text
localhost:5173
localhost:8000
localhost:5433
localhost:6380
npm run dev
```

这些只属于本地开发。

## 端口规范

| 端口 | 所属模式 | 用途 |
|------|----------|------|
| `5173` | 本地开发 | Vite 前端开发服务 |
| `8000` | 本地开发 | Express 后端开发服务 |
| `5433` | 本地开发/本地容器测试 | PostgreSQL 暴露到宿主机 |
| `6380` | 本地开发/本地容器测试 | Redis 暴露到宿主机 |
| `3780` | 本地容器测试/生产 | Web 对外访问入口 |

不要使用 `5175` 作为常规开发端口。除非 `5173` 被占用并且只是临时调试，否则统一使用 `5173`。

## 容器命名规范

本项目固定使用这些容器名：

```text
3dparthub-web
3dparthub-api
3dparthub-postgres
3dparthub-redis
```

不要再手动创建类似 `3dparthub-redis-6380` 的临时容器。现在 `docker-compose.local.yml` 已经把 Redis 映射到宿主机 `6380`。

如果历史上创建过临时 Redis 容器，确认不用后可以清理：

```bash
docker stop 3dparthub-redis-6380
docker rm 3dparthub-redis-6380
```

清理临时 Redis 不会影响 PostgreSQL 数据库。

## 常见问题

### 后端提示 Can't reach database server at localhost:5433

说明本地 PostgreSQL 没起来。执行：

```bash
colima start
docker compose -f docker-compose.local.yml up -d postgres
```

### 后端提示 Redis ECONNREFUSED localhost:6380

说明本地 Redis 没起来或没有映射端口。执行：

```bash
colima start
docker compose -f docker-compose.local.yml up -d redis
```

### 前端能打开但接口 502

通常是 Vite 代理目标不通、后端没启动，或后端连不上数据库。检查：

```bash
curl http://localhost:3780/api/health
curl http://localhost:5173/api/health
CHECK_VITE=1 node scripts/check-local-stack.mjs
docker ps
```

默认不要让 `5173` 直接代理 `8000`。只有确认本机后端 `server npm run dev` 正在运行时，才设置 `VITE_DEV_PROXY_TARGET=http://localhost:8000`。

### Docker 提示 docker.sock 不存在

说明 Colima 没启动。执行：

```bash
colima start
```

### 什么时候重启后端

如果先启动了后端，后启动数据库或 Redis，后端里可能保留旧连接错误。依赖服务启动后，重启一次后端最干净：

```bash
cd server
npm run dev
```

## 给后续维护者的约定

1. 本地开发默认先启动 `postgres` 和 `redis`，再启动后端和前端。
2. 本地前端固定使用 `5173`。
3. 本地后端固定使用 `8000`。
4. 本地 PostgreSQL 固定使用 `5433`。
5. 本地 Redis 固定使用 `6380`。
6. 生产环境只使用 `docker-compose.yml`，不使用本地 `.env` 的 `localhost` 地址。
7. 不要删除数据库卷，除非明确要清空环境并且已经备份。
