# CAD 模型在线浏览二次开发计划书

## 1. 项目背景

当前系统已经具备模型上传、分类、详情页预览、下载、分享、后台管理等业务能力。现有 3D 在线浏览方案采用 `occt-import-js / OpenCascade` 解析 STEP/IGES，后端生成 GLB/glTF，前端使用 Three.js 渲染。

当前主要问题不是业务系统缺失，而是 Viewer 层还没有达到企业级 CAD 浏览体验：

- 部分模型显示效果不稳定，存在明暗、材质、清晰度、边线、取景问题。
- 缩略图生成方式偏简化，复杂装配体容易出现线稿化、丢面感、模糊感。
- 现有 Viewer 工具能力较弱，缺少成熟 CAD Viewer 常见的测量、剖切、爆炸、零件隐藏、模型树、视图 cube 等能力。
- 当前实现是自研 Viewer，没有直接集成 Online 3D Viewer、Mayo 或完整 OpenCascade.js Web 应用。

## 2. 二次开发总体结论

推荐采用“三层组合”的二次开发路线：

```text
OpenCascade / occt-import-js
负责 CAD 文件解析、STEP/IGES 网格化、转换底座

Online 3D Viewer
作为 Web Viewer 实现参考，吸收其加载、交互、材质、相机、文件处理思路

Mayo
作为工程 CAD 功能参考，借鉴模型树、测量、剖切、爆炸、属性查看等能力设计
```

不建议直接整体嵌入 Online 3D Viewer，也不建议把 Mayo 作为 Web 前端嵌入。最佳策略是保留当前业务系统，重构内部 Viewer 能力。

## 3. 三个开源项目的角色定位

| 项目 | 定位 | 在本项目中的作用 | 是否直接集成 |
|---|---|---|---|
| OpenCascade / occt-import-js / OpenCascade.js | CAD 内核能力 | STEP/IGES 解析、网格化、未来几何计算 | 保留并增强 |
| OCCT WebGL Sample | 官方示例 | 参考 OCCT Web 展示方式 | 不直接集成 |
| Online 3D Viewer | Web 端开源 Viewer | 参考 Viewer 架构、文件加载、交互、材质、相机逻辑 | 选择性吸收 |
| Mayo | 桌面 CAD Viewer/转换器 | 参考 CAD 功能体验、模型树、测量、剖切、爆炸 | 功能设计参考 |

## 4. 目标架构

### 4.1 当前架构

```text
用户上传 STEP/IGES/XT
        ↓
后端 occt-import-js 解析
        ↓
生成 GLB/glTF + 缩略图
        ↓
前端 Three.js Viewer 加载
        ↓
模型详情页展示、下载、分享
```

### 4.2 目标架构

```text
上传 CAD 文件
        ↓
转换服务
  - OpenCascade / occt-import-js
  - 统一转换参数
  - 输出 GLB + meta.json + 转换诊断
        ↓
资产优化服务
  - glTF/GLB 压缩
  - 法线修复
  - 包围盒/零件树/材质信息
  - 真实 3D 缩略图
        ↓
企业级 Web Viewer
  - Three.js / react-three-fiber
  - Online 3D Viewer 思路重构加载和交互
  - Mayo 思路补 CAD 工具
        ↓
业务系统
  - 权限
  - 分类
  - 下载
  - 收藏
  - 分享
  - 后台管理
```

## 5. 功能规划

### 第一阶段：稳定当前转换与显示

目标：先让现有模型“稳定显示、清晰显示、缩略图正常”。

任务：

- 梳理 STEP/IGES/XT 转换链路，统一转 GLB。
- 保留原始 STEP 文件，GLB 仅作为前端预览资产。
- 修复缩略图生成逻辑，改为真实 3D 渲染截图或更接近前端 Viewer 的服务端渲染。
- 统一模型坐标、单位、包围盒、相机初始视角。
- 修复复杂装配体缩略图线稿化、过亮、过暗、取景偏小问题。
- 增加转换诊断日志：零件数、顶点数、面数、模型包围盒、转换耗时、失败原因。

验收标准：

- 普通 STEP 模型上传后可稳定生成 GLB。
- 缩略图不再出现明显透明线稿、主体过小、主体缺失。
- 模型详情页刷新、缩略图更新、重新转换不报错。
- 后端构建通过，前端构建通过。

### 第二阶段：Viewer 重构

目标：参考 Online 3D Viewer，把当前 Viewer 从“能看”提升到“好用”。

任务：

- 重构 `ModelViewer`，拆分为 Loader、Scene、Camera、Toolbar、Selection、Measure 等模块。
- 模型加载层统一支持 GLB/glTF，前端直接解析 STEP 只作为备用能力。
- 自动居中和自适应相机：按包围盒设置视角、near/far、缩放范围。
- 增加标准视图：正视、侧视、俯视、等轴测、后视、左视、右视。
- 增加视图 cube，替换或优化现有左下角/右下角轴向控件。
- 增加可配置的材质预设：默认灰、金属、塑料、透明。
- 增加可选边线显示，但默认关闭，避免模型模糊。
- 增加清晰模式：高对比背景、CAD 灰色材质、低强度环境光、适度主光。

验收标准：

- 页面首次进入时模型主体清晰居中。
- 不同尺寸模型相机不会过近、过远或裁切。
- 默认视觉效果稳定，不需要后台频繁调参。
- Viewer UI 不遮挡模型主体，桌面和移动端都可用。

当前进度（2026-04-27）：

- 已完成 Viewer 结构拆分：`ModelViewer`、`MultiFormatLoader`、`Scene`、`CameraController`、`CadViewerToolbar`、`ViewCube`、`ViewerTuningPanel` 已拆出。
- 已完成 GLB/glTF 优先加载，STEP/IGES 前端解析仅作为备用能力。
- 已完成自动居中、自适应相机、标准视图、视角盒、重置显示、截图、管理员诊断入口。
- 已完成材质预设：原色、智能灰、金属、塑料、玻璃；移动端工具栏也已补齐材质入口。
- 已完成原始材质保留：GLB/STEP 如果带颜色，前端可切到“原色”查看；默认仍使用更稳定的“智能灰”。
- 已完成边线默认关闭，避免模型发糊；需要时可手动开启实体边线。

### 第三阶段：CAD 工具能力

目标：参考 Mayo 增加工程浏览能力。

任务：

- 模型结构树：零件列表、搜索、隐藏/显示、隔离显示。
- 零件选择：点击零件高亮，但默认不弹出干扰面板。
- 测量工具：
  - 点到点距离
  - 边长
  - 半径/直径
  - 角度
  - 包围盒尺寸
- 剖切工具：
  - X/Y/Z 平面剖切
  - 剖切位置滑块
  - 一键关闭剖切
- 爆炸图：
  - 按零件包围盒中心计算方向
  - 爆炸比例滑块
  - 一键恢复
- 属性面板：
  - 模型名称
  - 格式
  - 文件大小
  - 零件数
  - 面数
  - 顶点数
  - 上传时间

验收标准：

- 管理员和普通用户都能稳定浏览模型。
- 工具栏清晰，不影响拖拽、旋转、缩放。
- 点击零件、隐藏零件、恢复显示不卡顿。
- 测量、剖切、爆炸功能互不冲突。

当前进度（2026-04-27）：

- 已开始 P2：新增隐藏式模型结构面板，默认不显示，不影响旋转、缩放、拖拽。
- 已完成零件索引：前端从 GLB/STEP 加载后的 mesh 中提取零件名称、路径、顶点数、面数。
- 已完成零件搜索、点击选中、高亮边框、隐藏/显示、隔离显示、全部显示。
- 已完成桌面端和移动端工具栏入口，移动端结构面板避开右侧工具栏。
- 已保留“点击零件不弹出干扰面板”的交互，只做高亮选中。
- 已完成测量工具第一版：两点距离测量、模型包围盒尺寸查看、测量线/点/距离标签渲染。
- 已处理测量与零件选择冲突：测量打开时点击模型采集测量点，关闭后恢复零件选中。
- 当前已进入 P3：剖切工具已支持 X/Y/Z 方向、位置滑块、反向剖切、归中重置，桌面端和移动端都可操作。
- P3 持续完善：测量工具已补充三点角度、三点直径；爆炸图已支持按模型尺寸计算偏移、爆炸比例滑块和一键恢复。
- P3 属性面板已补齐：普通用户可在模型浏览页查看格式、文件大小、上传时间、包围盒、零件数、顶点数、面数；选中零件后可查看该零件统计。
- P3 测量点已改为保守模式：先保留稳定的表面点测量，关闭测量工具时立即清空覆盖点；顶点/边吸附后续改为独立可开关能力再上线。
- P3 测量工具继续完善：已增加 mm/cm/m/自动单位切换、测量结果复制，并修正测量提示为稳定表面点模式。
- P3 测量历史已补充：距离、角度、直径测完后会保留最近 12 条记录，支持单条复制、删除、全部清空，并在 Viewer 内以淡色标注保留显示。
- P3 测量吸附已补充：测量工具支持表面、边、顶点三种吸附模式，默认表面稳定模式，边/顶点吸附基于当前命中三角面计算，避免大模型全局遍历。
- P3 剖切工具继续完善：剖切开启后 Viewer 内会显示半透明切面、边框、方向短线和当前位置标签，便于定位切面位置。
- P3 剖切交互继续完善：剖切面支持在 Viewer 内直接拖动，拖动时按当前剖切范围自动夹取位置，仍保留右侧滑块精调。

### 第四阶段：性能与大模型优化

目标：保证长期大量模型和复杂装配体下可用。

任务：

- 引入 glTF 优化流程：
  - meshoptimizer
  - Draco 或 Meshopt 压缩
  - 减少重复材质
  - 去除无效节点
- 为大模型生成 LOD 或简化预览版本。
- 对 GLB、缩略图、meta.json 使用缓存和版本号。
- 加载进度显示：文件下载、解析、GPU 上传分阶段展示。
- 增加模型预检查：超大模型提示、转换失败重试、异常文件隔离。

验收标准：

- 大模型加载有明确进度，不白屏。
- 常规模型详情页加载速度明显提升。
- 转换失败不会影响其它任务。
- 缓存更新后前端不会显示旧缩略图。

当前进度（2026-04-27）：

- 已开始 P4：服务端转换 GLB 时会自动将可降级的索引缓冲写为 `Uint16`，减少中小零件的索引体积；预览诊断会记录 GLB 大小、原文件大小、体积比例、索引优化收益和大模型性能提示。
- 已开始 P4：前端模型加载遮罩从单一百分比改为“准备模型 / 下载模型 / 解析几何 / 上传 GPU”分阶段提示，便于判断卡在下载还是解析阶段。

### 第五阶段：企业级体验完善

目标：做出接近 eDrawings/CADENAS 的业务体验，但保持低授权成本。

任务：

- Viewer 工作区布局升级：
  - 顶部工具栏
  - 右侧属性/结构面板
  - 底部状态栏
  - 快捷视图按钮
- 管理员调试模式：
  - 当前模型渲染信息
  - 材质/灯光临时调试
  - 保存为全局默认配置
- 失败模型诊断页面：
  - 原文件信息
  - 转换日志
  - GLB 检查结果
  - 缩略图生成结果
- 分享页轻量 Viewer：
  - 保留旋转、缩放、视图切换
  - 默认隐藏高级调试功能

验收标准：

- 模型浏览页达到企业产品库视觉标准。
- 管理员可以快速定位模型显示问题。
- 普通用户使用时界面简洁，不被调试功能干扰。

## 6. 技术实现建议

### 6.1 后端转换

保留当前 `server/src/services/converter.ts`，逐步增强：

- `convertStepToGltf()` 继续使用 `occt-import-js`。
- 输出 GLB 作为主预览格式。
- 生成 `{modelId}.meta.json` 保存零件、包围盒、面数、顶点数、材质信息。
- 增加转换参数配置：
  - `linearDeflection`
  - `angularDeflection`
  - 是否保留颜色
  - 是否合并小零件
  - 是否生成边线数据

### 6.2 前端 Viewer

保留 `Three.js + react-three-fiber`，但参考 Online 3D Viewer 重构内部结构：

```text
components/3d/
  ModelViewer.tsx
  MultiFormatLoader.tsx
  Scene.tsx
  CameraController.tsx
  ViewCube.tsx
  CadViewerPanel.tsx
  CadViewerToolbar.tsx
  PreviewDiagnosticsDialog.tsx
```

### 6.3 缩略图

建议从当前 2D canvas 三角面绘制，升级为真实渲染截图：

```text
GLB
 ↓
服务端 headless renderer
 ↓
加载同一套材质/灯光/相机
 ↓
输出 PNG/WebP 缩略图
```

这样缩略图会和在线 Viewer 更一致，避免出现“预览图像线稿、模型详情正常”的割裂。

## 7. 授权与合规注意

- Online 3D Viewer 通常按 MIT 开源授权使用，适合参考和二次开发，但复制代码前需要保留许可声明。
- Mayo 使用较宽松的开源授权，适合作为功能设计和算法参考；如果直接移植代码，需要保留许可声明。
- OpenCascade.js / OCCT 相关授权需要单独核对版本和依赖，尤其是 LGPL 动态链接、WebAssembly 分发、商业闭源部署边界。
- 本项目如果作为商业系统发布，应建立 `THIRD_PARTY_NOTICES.md`，记录所有开源组件和许可证。

## 9. 风险清单

| 风险 | 影响 | 应对 |
|---|---|---|
| STEP 文件来自不同 CAD 软件，质量差异大 | 转换失败或丢面 | 建立失败样本库，记录转换参数和失败原因 |
| 复杂装配体面数过高 | 加载慢、浏览器卡顿 | GLB 压缩、LOD、懒加载、零件合并策略 |
| 前端直接处理 STEP 过重 | 首屏慢、内存高 | 主链路改为服务端转 GLB |
| 边线显示导致模糊 | 视觉效果下降 | 默认关闭边线，仅管理员/工具栏启用 |
| 缩略图与 Viewer 不一致 | 用户误判模型质量 | 缩略图改为真实渲染截图 |
| 开源许可证边界不清 | 商业发布风险 | 建立第三方许可证清单，必要时做法律审核 |

## 10. 最终推荐路线

最终路线不是替换成某一个开源项目，而是组合使用：

```text
底层：OpenCascade / occt-import-js
Viewer 参考：Online 3D Viewer
CAD 功能参考：Mayo
业务系统：保留当前模型库平台
```

这是长期费用最低、可控性最高、最适合当前系统的路线。
